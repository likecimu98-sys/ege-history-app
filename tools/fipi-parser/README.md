# Парсер открытого банка заданий ФИПИ

Локальный инструмент с веб-интерфейсом: выкачивает задания с ege.fipi.ru
по выбранному предмету, темам и типу ответа, сохраняет их в JSON + HTML
вместе с картинками.

## Как пользоваться

1. Дважды кликните **«Запустить парсер ФИПИ.bat»** — откроется браузер.
2. Выберите предмет (по умолчанию — История).
3. Отметьте нужные разделы/темы и тип ответа (или ничего — тогда всё).
4. При желании поставьте галочку **«Также подобрать ответы»** (см. ниже).
5. «Найти задания» → «Выкачать все найденные» → «Сохранить на компьютер».
6. Результат появится в папке `output/`:
   - `задания.json` — полная выгрузка (сырой html + разбор + ответы);
   - `банк.json` — чистый нормализованный банк для импорта в программы;
   - `задания.html` — открыть в браузере, читать, печатать (с ответами);
   - `images/` — все картинки заданий.

Чтобы остановить парсер — закройте чёрное окно.

## Подбор ответов

Банк ФИПИ не публикует ответы напрямую: правильность проверяет их сервер
(`solve.php`), возвращая только «верно/неверно». Поэтому ответ **подбирается
перебором** — программа шлёт варианты, пока сервер не подтвердит верный.

- Работает для типов с закрытым ответом: **установление соответствия,
  расстановка терминов, последовательность, выбор нескольких**.
- НЕ работает для **краткого текстового ответа** (слово/имя) и **эссе** —
  их пространство ответов бесконечно; такие задания сохраняются без ответа.
- Это **медленно** (соответствие из 4 позиций ≈ 20–30 сек) и нагружает сервер
  ФИПИ — перебор идёт с паузами, лимит 800 попыток на задание. Для большой
  темы (60+ заданий) закладывайте 20–40 минут. Галочка по умолчанию выключена.
- Найденный ответ показывается и в интерфейсе, и во всех файлах: `answerRaw`
  (строка ФИПИ, напр. `512`) и `answer` (расшифровка: «А — 5 (Будённый)…»).

## Важно

- Сайт ФИПИ блокирует зарубежные IP: если включён VPN на весь компьютер,
  задания могут не загружаться — отключите VPN на время выкачивания.
- Задания принадлежат ФГБНУ «ФИПИ» — используйте для личной подготовки
  и занятий с учениками, при публикации указывайте источник.

## Технические детали

- `server.js` — Node.js (18+), без зависимостей. Ходит на ege.fipi.ru напрямую
  (Node не использует системный прокси), декодирует windows-1251, разбирает
  блоки `.qblock`, вытаскивает картинки из вызовов `ShowPictureQ*` и проксирует
  их через `/fipi-file`.
- Фильтр отправляется POST-ом на `questions.php` (поля `theme`, `qkind`, `qid`
  через запятую), дальше ФИПИ хранит его в сессии — страницы листаются GET-ом
  с cookie. Общее число заданий приходит в `setQCount(N)`.
- GUID предметов парсятся с главной `/bank/`; История ЕГЭ —
  `068A227D253BA6C04D0C832387FD0D89`.
- Подбор ответов: форма ответа разбирается из задания (`select ans0..ansK` →
  цепочка цифр; `test0..testN` → битовая маска; radio → индекс). Кандидаты
  перебираются (сначала перестановки без повторов — реальные ответы почти
  всегда такие) и шлются на `solve.php`; вердикт `3` = верно, `2` = неверно.
  Расшифровка берёт пронумерованные «Пропущенные элементы» из текста задания.
