# Парсер открытого банка заданий ФИПИ

Локальный инструмент с веб-интерфейсом: выкачивает задания из открытого банка
ФИПИ (**ЕГЭ — ege.fipi.ru** и **ОГЭ — oge.fipi.ru**) по выбранному экзамену,
предмету, темам и типу ответа, сохраняет их в JSON + HTML вместе с картинками.

## Как пользоваться

1. Дважды кликните **«Запустить парсер ФИПИ.bat»** — откроется браузер.
2. Вверху выберите экзамен: **ЕГЭ** или **ОГЭ** (список предметов и тем
   меняется под выбранный экзамен).
3. Выберите предмет (по умолчанию — История).
4. Отметьте нужные разделы/темы и тип ответа (или ничего — тогда всё).
5. При желании поставьте галочку **«Также подобрать ответы»** (см. ниже).
6. «Найти задания» → «Выкачать все найденные» → «Сохранить на компьютер».
7. Результат появится в папке `output/`:
   - `задания.json` — полная выгрузка (сырой html + разбор + ответы);
   - `банк.json` — чистый нормализованный банк для импорта в программы;
   - `задания.html` — открыть в браузере, читать, печатать (с ответами);
   - `images/` — все картинки заданий.

Если сервер или ФИПИ оборвал длинную загрузку, снова запустите парсер и
нажмите **«Продолжить с места остановки»**. Программа хранит локальный
чекпойнт: уже скачанные задания и найденные ответы повторно не обрабатываются.

Задания по одному общему материалу (карта, письменный источник, изображение)
загружаются всей группой. Карта или текст выводятся один раз перед связанными
вопросами. Если в уже сохранённой выгрузке общего материала нет, откройте
**«Конструктор заданий»** и нажмите **«Повторить загрузку карт и текстов»**.
Программа допишет найденные материалы в существующую выгрузку; скачивать все
задания заново не нужно. После сбоя или остановки повторное нажатие продолжит
работу только с оставшимися группами.

Если после полной загрузки ФИПИ временно не отдал часть карт, перед сохранением
появится кнопка **«Повторить загрузку карт»**. Она повторно запрашивает только
неполные блоки 9–12; задания и уже найденные ответы заново не скачиваются.

Чтобы остановить парсер — закройте чёрное окно.

## Подбор ответов

Банк ФИПИ не публикует ответы напрямую: правильность проверяет их сервер
(`solve.php`), возвращая только «верно/неверно». Поэтому ответ **подбирается
перебором** — программа шлёт варианты, пока сервер не подтвердит верный.

- Работает для типов с закрытым ответом: **установление соответствия,
  расстановка терминов, последовательность, выбор нескольких**.
- НЕ работает для **краткого текстового ответа** (слово/имя) и **эссе** —
  их пространство ответов бесконечно; такие задания сохраняются без ответа.
- Это **медленно** (соответствие из 4 позиций ≈ 20–30 сек) и нагружает сервер
  ФИПИ — перебор идёт с паузами, лимит 800 попыток на задание. Для большой
  темы (60+ заданий) закладывайте 20–40 минут. Галочка по умолчанию выключена.
- Найденный ответ показывается и в интерфейсе, и во всех файлах: `answerRaw`
  (строка ФИПИ, напр. `512`) и `answer` (расшифровка: «А — 5 (Будённый)…»).

## Важно

- Сайт ФИПИ блокирует зарубежные IP: если включён VPN на весь компьютер,
  задания могут не загружаться — отключите VPN на время выкачивания.
- Задания принадлежат ФГБНУ «ФИПИ» — используйте для личной подготовки
  и занятий с учениками, при публикации указывайте источник.

## Технические детали

- `server.js` — Node.js (18+), без зависимостей. Ходит на ege.fipi.ru и
  oge.fipi.ru напрямую (Node не использует системный прокси), декодирует
  windows-1251, разбирает блоки `.qblock`, вытаскивает картинки из вызовов
  `ShowPictureQ*` и проксирует их через `/fipi-file?...&exam=ege|oge`.
- ЕГЭ и ОГЭ — один и тот же движок ФИПИ, отличаются только хостом. Параметр
  `exam` (`ege`/`oge`) прокидывается во все API и выбирает `examHost()`; cookie
  сессии хранится отдельно на каждый хост (`cookieJars[hostname]`).
- Фильтр отправляется POST-ом на `questions.php` (поля `theme`, `qkind`, `qid`
  через запятую), дальше ФИПИ хранит его в сессии — страницы листаются GET-ом
  с cookie. Общее число заданий приходит в `setQCount(N)`.
- Для блоков `.number-in-group` парсер берёт `zid`, отдельно запрашивает общий
  материал группы и сохраняет `stimulusHtml`, `stimulusText` и
  `stimulusImages`. Перед продолжением пагинации исходный фильтр сессии
  восстанавливается.
- Прогресс длинной операции хранится в `output/_last-session.json`; после
  успешного сохранения выгрузки чекпойнт удаляется.
- GUID предметов парсятся с главной `/bank/`; История ЕГЭ —
  `068A227D253BA6C04D0C832387FD0D89`, История ОГЭ —
  `3CBBE97571208D9140697A6C2ABE91A0`.
- Подбор ответов: форма ответа разбирается из задания (`select ans0..ansK` →
  цепочка цифр; `test0..testN` → битовая маска; radio → индекс). Кандидаты
  перебираются (сначала перестановки без повторов — реальные ответы почти
  всегда такие) и шлются на `solve.php`; вердикт `3` = верно, `2` = неверно.
  Расшифровка берёт пронумерованные «Пропущенные элементы» из текста задания.
